# Update Protobuf Interfaces Implementation Plan

## Overview
This plan outlines the steps to update the `AiService` protobuf definitions to include more context and feedback for browser actions. This will enable more intelligent decision-making by the AI service.

## Scope Definition (CRITICAL)
### In Scope
- Define `Link`, `Button`, `FormField`, and `Form` messages in `proto/ai/v1/ai.proto`.
- Update `DecideBrowserActionRequest` with new fields: `last_action_success`, `last_action_error`, `current_url`, `page_title`, `links`, `buttons`, `forms`.
- Run `task proto` to generate stubs for Go, TypeScript, and Python.
- Verify the presence of new fields in generated files.

### Out of Scope (DO NOT TOUCH)
- Implementing the logic to populate these fields in the orchestrator.
- Implementing the extraction logic in the browser service.
- Updating the AI server logic to use these new fields.

## Current State Analysis
- `proto/ai/v1/ai.proto`: `DecideBrowserActionRequest` has 3 fields (1-3).
- `Taskfile.yml`: `proto` task handles all generation.

- `ui/src/gen/ai/v1/ai_pb.ts`: TypeScript stubs.
