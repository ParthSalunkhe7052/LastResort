# Fix Duplicate SQLi Execution Implementation Plan

## Overview
Consolidate `sqli_basic` and `sqli_agent` into a single `sqli_agent` module. This module will first execute static payloads (deterministic fallback) before proceeding with AI-driven testing. This prevents duplicate findings and reduces target load. Additionally, a browser health check will be added to the scan start.

## Scope Definition (CRITICAL)
### In Scope
- Move deterministic SQLi verification logic to a shared location.
- Integrate static SQLi payloads into `AgentSQLiExecutor`.
- Remove `ModuleSqliBasic` from scan profiles and orchestrator dispatch.
- Add browser health check at scan start.
- Update `orchestrator_test.go` to reflect module changes.
### Out of Scope (DO NOT TOUCH)
- Changing other vulnerability modules (XSS, CSRF, etc.).
- Modifying the AI service implementation itself.

- `internal/orchestrator/phase.go:27`: `ProfileModules` includes both `sqli_basic` and `sqli_agent`.
